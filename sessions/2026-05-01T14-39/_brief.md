# 📋 작업 브리프

**원 명령:** 이 내용을 바탕으로 웹사이트를 만들어보자

## 요약
기존에 확정된 '콜렉(Collect)' MVP 계획을 기반으로 실제 사용자에게 보여줄 웹사이트를 구축하는 단계입니다. 단순히 코딩에 앞서, 웹사이트의 전체적인 구조(와이어프레임), 콘텐츠(카피), 그리고 기술 아키텍처를 동시에 확정하는 작업이 필요합니다.

## 분배
- **📱 Secretary**: 프로젝트의 웹사이트 목표와 사용자 흐름(User Flow)을 정의하는 종합 브리핑을 작성하십시오. 메인 랜딩 페이지(Landing Page), 사용자 대시보드(Dashboard), 그리고 핵심 기능인 '콜렉(Collection) 생성/조회' 페이지의 3가지 핵심 페이지 구성을 중심으로 전체 사이트맵과 필요한 기능을 목록화하여 전달하세요.
- **🎨 Designer**: Secretary가 제공한 사이트맵과 기존의 '따뜻함, 기억'을 컨셉으로 하는 디자인 시스템을 적용하여, 1) 메인 랜딩 페이지(최상단부터 스크롤 다운되는 흐름), 2) 사용자 대시보드, 3) 콜렉 생성/조회 페이지의 와이어프레임(고해상도)을 제작하십시오. 특히, 관계의 심리적 가치를 시각화하는 요소(예: 관계 그래프, 타임라인)를 중심으로 디자인을 전개해야 합니다.
- **✍️ Writer**: 웹사이트의 주요 페이지(랜딩 페이지, 기능 설명 섹션)에 들어갈 핵심 카피라이팅을 작성하십시오. 타깃 청중(기념일 준비에 고민하는 손님/호스트)의 'Pain Point'를 건드리고, 'Collect'가 제공하는 감성적 해결책을 강조하는 헤드라인, 서브 카피, 그리고 명확한 CTA(Call to Action) 문구를 작성해주세요.
- **💻 Developer**: Designer와 Writer가 확정한 와이어프레임과 콘텐츠를 바탕으로, 웹사이트의 초기 기술 스택(Front-end/Back-end)을 최종 확정하고, API 엔드포인트(Endpoints)를 구체적으로 정의하십시오. 특히, 관계 데이터(Relationship Graph)를 처리하기 위한 데이터 파이프라인의 초기 구조와 필수 DB 테이블 관계를 설계하여 개발 착수 준비를 완료하십시오.
